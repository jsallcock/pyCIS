Camera¶

class pycis.model.camera.Camera(bit_depth, sensor_format, pixel_size, qe, epercount, cam_noise, mode='mono')[source]¶

Camera class.

Parameters:
  • bit_depth (int) – Bit depth of camera pixels.
  • sensor_format (tuple) – Number of pixels in each dimension (x, y, ).
  • pix_size (float) – Pixel size in m.
  • qe (float) – Quantum efficiency or sensor in units e- per photon.
  • epercount (float) – (float) Conversion gain of sensor in units e- per count.
  • cam_noise (float) – (float) Camera noise standard deviation in units e-.
  • mode (str) – Describes the mode of sensor operation. Currently supported are: ‘mono’ for monochrome, ‘rgb’ (untested) for color imaging and ‘mono_polarised’ for monochrome with a pixelated polariser array (layout of the FLIR Blackfly S camera).
capture(spectrum, apply_polarisers=None, clean=False)[source]¶

Capture an image.

Parameters:
  • spectrum (xr.DataArray) – Spectrum in units of photons / m with dimensions ‘x’, ‘y’, ‘wavelength’ and (optionally) ‘stokes’. If there is no ‘stokes’ dim then the light is assumed to be unpolarised.
  • apply_polarisers (bool) – Whether to apply a pixelated polariser array to the spectrum.
  • clean (bool) – False to add realistic image noise. Clean images used for testing.
Returns:

(xr.DataArray) Captured image.
get_pixel_position(x_pixel=None, y_pixel=None)[source]¶

Calculate pixel positions (in m) on the camera’s sensor plane (the x-y plane).

The origin of the x-y coordinate system is the centre of the sensor. Pixel positions correspond to the pixel centres. If x_pixel and y_pixel are specified then only returns the position of that pixel. :param x_pixel: :param y_pixel: :return: x_pos, y_pos, both instances of xr.DataArray
