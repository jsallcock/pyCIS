Instrument¶

class pycis.model.instrument.Instrument(camera, optics, interferometer, force_mueller=False)¶

Coherence imaging instrument

get_instrument_type()¶

For certain instrument layouts there are analytical shortcuts for calculating the interferogram, skipping the full Mueller matrix treatment.

current instrument types: - ‘mueller’: Full Mueller calculation - ‘single_delay_linear’ - ‘single_delay_polarised’ - ‘multi_delay_polarised’

Returns

itype (str)

capture(spectrum, clean=False)¶

capture image of scene

Parameters

spectrum – (xr.DataArray) photon fluence spectrum with units of ph / m [hitting the pixel area during

exposure time] and with dimensions ‘wavelength’, ‘x’, ‘y’ and (optionally) ‘stokes’. If no stokes dim then it is assumed that light is unpolarised (i.e. the spec supplied is the S_0 Stokes parameter only)

Parameters

clean – (bool) False to add realistic image noise, passed to self.camera.capture()

Returns

get_delay(wavelength)¶

calculate the interferometer delay(s) at the given wavelength(s)

At the moment this method only works for instrument types other than ‘mueller’. I’m not sure it would be possible to write a general function?

Parameters

wavelength – in units (m), can be a float or an xr.DataArray with dimensions including ‘x’ and ‘y’

Returns

delay in units (rad)

get_fringe_frequency(wavelength)¶

calculates the (rough) interference fringe period at the sensor plane and at the given wavelength

only makes sense for instrument types with a phase shear (e.g. ‘single_delay_linear’ and ‘multi_delay_polarised’)
